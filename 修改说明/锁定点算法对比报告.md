# 锁定点算法对比报告

> 对比 `get_lock_point`（原算法）与 `get_lock_point_by_peak_valley_pairing`（新增算法）

---

## 1. 核心差异：锁定点定位策略

| 维度 | `get_lock_point`（原算法） | `get_lock_point_by_peak_valley_pairing`（新算法） |
|------|---------------------------|--------------------------------------------------|
| **策略** | 单一峰谷极值对 | 多峰多谷全局寻峰 + 智能配对 |
| **特征检测** | `argmin`/`argmax` 各取一个 | `scipy.find_peaks` 检测所有满足阈值的峰谷 |
| **阈值过滤** | 无（只用最值） | 峰高度 > 最大绝对值的 **1/2** |
| **配对逻辑** | 自然配对（唯一的 min/max） | 贪心配对（按绝对值排序，方向感知） |

---

## 2. 算法流程对比

### 原算法 `get_lock_point`

位置：`linien-common/linien_common/common.py:212-266`

```
1. 截取 [x0, x1) 数据
2. min_idx = argmin, max_idx = argmax  → 只有一对
3. mean = (min+max)/2
4. 锁定点 = 均值去零后 |signal| 最小的位置
```

**特点**：简单直接，假设用户选定区域内只有一对有意义的峰谷。

### 新算法 `get_lock_point_by_peak_valley_pairing`

位置：`linien-common/linien_common/common.py:269-434`

```
1. 截取 [x0, x1) 数据
2. 阈值 = max(|signal|) / 2
3. find_peaks(signal)   → 所有峰（高度 > 阈值）
4. find_peaks(-signal)  → 所有谷（|高度| > 阈值）
5. 峰、谷分别按绝对值降序排序
6. 贪心配对（谷按绝对值从大到小遍历）：
   - 谷在最强候选峰之前 → 取谷之后第一个峰（索引最小者）
   - 谷在最强候选峰之后 → 取谷之前最后一个峰（索引最大者）
   - 每个峰只能与一个谷配对
7. 选第一个配对（最强谷对应），求过零点
```

**特点**：复杂稳健，能处理多峰干扰、噪声、不对称信号。

---

## 3. 相同点

- **返回值结构完全一致**：`(mean_signal, target_slope_rising, target_zoom, rolled_error_signal, line_width, peak_idxs)`
- **`target_zoom` 计算方式相同**：`N_POINTS / 峰谷距离 / final_zoom_factor`
- **滚动对齐逻辑一致**：让锁定点居中，便于后续相关性匹配
- **过零点检测方式一致**：`argmin(|signal - mean|)`

---

## 4. 适用场景

| 场景 | 推荐算法 | 原因 |
|------|----------|------|
| 干净的单峰谷信号 | `get_lock_point` | 快速、无需 scipy 依赖 |
| 多峰干扰（如法布里-珀罗多模） | **新算法** | 阈值过滤 + 配对避免误锁定到次级峰 |
| 噪声较大的信号 | **新算法** | 寻峰算法对噪声更鲁棒（需调阈值） |
| 强不对称信号 | **新算法** | 不依赖单一全局极值 |

---

## 5. 性能对比

| 指标 | `get_lock_point` | 新算法 |
|------|------------------|--------|
| 时间复杂度 | O(n) | O(n log n)（排序） |
| 依赖 | 仅 numpy | numpy + scipy |
| 实时性 | 适合实时 | 适合初始化阶段 |

---

## 6. 退化兼容性

新算法在以下情况自动退化为原算法（位于 `common.py:309-311` 和 `common.py:330-331`）：

- 信号全为零（`max_abs == 0`）
- 区域内检测不到峰或检测不到谷

这保证了向后兼容性。

---

## 7. 配对算法细节

新算法的配对逻辑（位于 `common.py:341-380`）：

```
对每个谷（按绝对值从大到小）：
    候选峰 = 所有未配对的峰
    最强候选峰 = 候选中绝对值最大者
    
    if 谷位置 < 最强候选峰位置:
        # 谷在峰之前 → 向后找
        候选 = 位置 > 谷位置 的所有未配对峰
        配对峰 = 候选中位置最小者（最靠近谷）
    else:
        # 谷在峰之后 → 向前找
        候选 = 位置 < 谷位置 的所有未配对峰
        配对峰 = 候选中位置最大者（最靠近谷）
    
    标记该峰为已配对
```

**设计意图**：
- 优先匹配最强的谷（最显著的特征）
- 每次匹配最近的可用峰，避免跨区域错配
- 每个峰只用一次，保证配对唯一性

## 8. 测试验证

测试文件：`tests/test_peak_valley_pairing.py`

已验证场景：
1. **基本返回类型** - 6 元组结构正确
2. **峰在前谷在后** - 锁定点位于两者之间
3. **谷在前峰在后** - 同样正确定位
4. **多峰多谷** - 正确选择最强配对
5. **阈值过滤** - 弱峰（低于阈值）不被误检测
