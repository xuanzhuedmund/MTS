# 多锁定点显示功能修改说明

> 版本：1.0
> 日期：2026-07-01

---

## 1. 修改目的

当选择"峰谷配对法"作为锁定点算法时，在绘图区域显示**所有峰谷对的锁定点位置**，方便用户预览所有可能的锁定位置。实际锁定时仍使用最强峰谷对的锁定点。

### 显示效果

| 算法 | 显示内容 |
|------|----------|
| 极值法（默认） | 只显示主锁定目标线（白色实线） |
| 峰谷配对法 | 主锁定目标线 + 所有辅助锁定点（黄色虚线） |

---

## 2. 修改的文件清单

### 2.1 公共算法库

#### `linien-common/linien_common/common.py`

**修改内容**：新增辅助函数 `get_all_lock_points_by_peak_valley_pairing`（第 269-355 行）

```python
def get_all_lock_points_by_peak_valley_pairing(
    error_signal: np.ndarray, x0: int, x1: int
) -> List[int]:
```

**功能**：计算选定区域内所有峰谷对的锁定点位置

**算法流程**：
1. 对选定区域进行寻峰和寻谷（阈值 = 最大绝对值的 1/2）
2. 按绝对值高度降序排序
3. 贪心配对（与原算法一致）
4. 计算每个峰谷对的过零点作为锁定点
5. 返回所有锁定点索引列表（按配对强度降序）

**与原算法的区别**：
- 原函数 `get_lock_point_by_peak_valley_pairing` 只返回最强峰谷对的锁定点
- 新函数返回**所有**峰谷对的锁定点

#### 修改点 3：新增显示/隐藏方法（第 746-758 行）

```python
def show_all_lock_points(self, lock_points):
    """显示所有峰谷对的锁定点位置"""
    for i, line in enumerate(self.lock_point_lines):
        if i < len(lock_points):
            line.setValue(lock_points[i])
            line.setVisible(True)
        else:
            line.setVisible(False)

def hide_all_lock_points(self):
    """隐藏所有辅助锁定点线"""
    for line in self.lock_point_lines:
        line.setVisible(False)
```

## 3. 功能逻辑

### 3.1 显示流程

```
用户选择"峰谷配对法"算法
        │
        ▼
用户拖选谱线区域
        │
        ▼
PlotWidget.mouseReleaseEvent()
        │
        ├──► get_lock_point_by_peak_valley_pairing(...)
        │    返回：最强峰谷对的锁定参数（用于锁频）
        │
        ├──► get_all_lock_points_by_peak_valley_pairing(...)
        │    返回：所有峰谷对的锁定点索引列表
        │
        └──► show_all_lock_points(all_lock_points)
             │
             ├── 第1条虚线 → 第1强峰谷对的锁定点
             ├── 第2条虚线 → 第2强峰谷对的锁定点
             ├── ...
             └── 第N条虚线 → 第N强峰谷对的锁定点
```

### 3.2 视觉区分

| 元素 | 样式 | 含义 |
|------|------|------|
| 白色实线 | `lock_target_line` | 主锁定目标线（最强峰谷对） |
| 黄色虚线 | `lock_point_lines` | 辅助锁定点（其他峰谷对） |

### 3.3 锁定执行规则

- **实际锁频**：始终使用最强峰谷对（`pairs[0]`）的锁定点
- **显示用途**：辅助锁定点仅供参考，不参与实际锁频

---

## 4. 容量限制

绘图组件预分配了 **10 条**虚线，最多可同时显示 10 个锁定点。如果峰谷对超过 10 个，超出部分不显示（实际场景中极少出现）。

| **锁频逻辑** | 实际锁频仍使用最强峰谷对，行为不变 |
| **服务器端** | 无需修改，服务器端仍使用原函数 |
