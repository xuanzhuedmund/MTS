# 执行锁定复选框功能修改说明

> 版本：1.0
> 日期：2026-07-01

---

## 1. 修改目的

新增"执行锁定"复选框，提供一种预览模式：用户可以选择谱线区域，查看锁定点的计算结果（目标线显示），但不执行实际的锁频操作。

### 使用场景

| 场景 | 操作 | 效果 |
|------|------|------|
| 调试锁定点 | 取消勾选 | 只显示锁定目标线，不执行锁频 |
| 正常锁频 | 勾选（默认） | 完整执行锁频流程 |

### 2.3 绘图组件文件

#### `linien-gui/linien_gui/ui/plot_widget.py`

**修改内容**：调整 `mouseReleaseEvent` 方法中自动锁频的调用逻辑

**关键变更**：将 `get_lock_point` 调用移到 `start_autolock` 之前，并根据 `autolock_enabled` 参数决定是否发送锁频命令

**修改前**：
```python
# 先调用 start_autolock，再计算锁定点用于预览
self.control.start_autolock(...)
(get_lock_point 结果) → self.autolock_ref_spectrum
```

**修改后**：
```python
# 先计算锁定点
(get_lock_point 结果) → self.autolock_ref_spectrum

# 根据 autolock_enabled 决定是否执行锁频
if self.parameters.autolock_enabled.value:
    self.control.start_autolock(...)
```

**代码位置**：第 310-333 行

## 3. 功能逻辑

### 3.1 完整调用链

```
用户点击"选择目标谱线"按钮
        │
        ▼
LockingPanel.start_autolock_selection()
        │
        ▼
parameters.autolock_selection = True
        │
        ▼
用户在图像上拖选区域
        │
        ▼
PlotWidget.mouseReleaseEvent()
        │
        ├──► 调用 get_lock_point / get_lock_point_by_peak_valley_pairing
        │    返回：rolled_error_signal（锁定点居中的误差信号）
        │
        ├──► self.autolock_ref_spectrum = rolled_error_signal
        │    （用于后续显示锁定目标线）
        │
        └──► 检查 parameters.autolock_enabled.value
             │
             ├── True → self.control.start_autolock(...)
             │          （发送 RPC 到服务器，执行完整锁频流程）
             │
             └── False → 不调用 start_autolock
                         （仅显示目标线，不执行锁频）
```

### 3.2 未勾选时的行为

当取消勾选"执行锁定"复选框时：

1. **锁定点计算**：仍然会调用 `get_lock_point` 算法计算锁定点
2. **目标线显示**：`autolock_ref_spectrum` 被更新，目标线会正常显示在图像上
3. **锁频命令**：不会调用 `control.start_autolock()`，因此不会触发服务器端的锁频流程
4. **后续影响**：扫描继续运行，PID 控制器不会被激活

## 5. 测试验证

### 5.1 测试步骤

1. ✅ 勾选"执行锁定"复选框，选择谱线区域 → 执行完整锁频流程
2. ✅ 取消勾选"执行锁定"复选框，选择谱线区域 → 只显示目标线，不执行锁频
3. ✅ 取消勾选状态下，选择区域后扫描继续运行
4. ✅ 取消勾选状态下，目标线正常显示和更新

| 测试用例 | 预期结果 |
|----------|----------|
