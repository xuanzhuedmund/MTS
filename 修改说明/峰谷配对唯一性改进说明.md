# 峰谷配对唯一性改进说明

> 版本：1.0
> 日期：2026-07-01

---

## 1. 修改目的

改进峰谷配对算法，确保**每个峰和每个谷至多参与一次配对**（双向唯一），同时消除不合理的跨区域强制配对。

### 背景问题

原算法存在两个问题：
1. **缺少显式的谷使用标记**：虽然外层循环天然保证每个谷只遍历一次，但缺少显式标记，代码意图不清晰
2. **退化分支不合理**：当某侧无候选峰时，原算法从全局剩余候选中选择"绝对值最接近"的峰进行**强制配对**，可能导致物理上不相关的峰谷被配对

## 2. 修改的文件清单

### 2.1 公共算法库

#### `linien-common/linien_common/common.py`

#### 修改点 1：新增公共配对函数 `_pair_peaks_valleys`（第 358-388 行）

提取配对逻辑为独立函数，统一被两个函数调用：

```python
def _pair_peaks_valleys(
    peaks_sorted: List[Tuple[int, float]],
    valleys_sorted: List[Tuple[int, float]],
) -> List[Tuple[Tuple[int, float], Tuple[int, float]]]:
```

**关键改进**：

| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 峰使用标记 | `used_peaks` | `used_peaks`（保留） |
| 谷使用标记 | 无（隐式） | `used_valleys`（显式） |
| 退化分支处理 | 从全局选绝对值最接近的峰 | **跳过该谷**（`continue`） |

#### 修改点 2：简化 `get_lock_point_by_peak_valley_pairing`

删除内联的配对逻辑，改为调用 `_pair_peaks_valleys`：

```python
# 修改前（40 行内联配对逻辑）
used_peaks = [False] * len(peaks_sorted)
pairs = []
for valley in valleys_sorted:
    ...

# 修改后（1 行调用）
pairs = _pair_peaks_valleys(peaks_sorted, valleys_sorted)
```

#### 修改点 3：简化 `get_all_lock_points_by_peak_valley_pairing`

同样改为调用统一的 `_pair_peaks_valleys` 函数。

#### 修改点 4：更新文档字符串

更新算法流程说明，明确标注双向唯一性：

```
3. 对所有峰点和谷点按绝对值高度由大到小排序，依次进行配对：
   - 谷按绝对值从大到小遍历，每个谷在剩余未配对峰中寻找最近的峰；
   - 若谷在峰之前，取谷之后最近的峰；若谷在峰之后，取谷之前最近的峰；
   - 每个峰和每个谷至多参与一次配对（双向唯一）；
```

## 3. 配对算法对比

### 3.1 原算法流程

```
遍历每个谷（按绝对值降序）
    │
    ├── 在剩余未配对峰中寻找候选
    │
    ├── 判断谷与最强候选峰的相对位置
    │   ├── 谷在峰之前 → 向后找最近的峰
    │   │   └── 若无候选 → 从全局选绝对值最接近的峰  ← 问题点
    │   │
    │   └── 谷在峰之后 → 向前找最近的峰
    │       └── 若无候选 → 从全局选绝对值最接近的峰  ← 问题点
    │
    └── 标记峰为已用（无谷标记）
```

### 3.2 改进后流程

```
遍历每个谷（按绝对值降序）
    │
    ├── 检查谷是否已使用 → 是则跳过
    │
    ├── 在剩余未配对峰中寻找候选
    │
    ├── 判断谷与最强候选峰的相对位置
    │   ├── 谷在峰之前 → 向后找最近的峰
    │   │   └── 若无候选 → 跳过此谷（不强制配对）  ← 改进点
    │   │
    │   └── 谷在峰之后 → 向前找最近的峰
    │       └── 若无候选 → 跳过此谷（不强制配对）  ← 改进点
    │
    └── 标记峰和谷均为已用  ← 改进点
```

## 4. 行为差异示例

### 场景：2 峰 1 谷，谷在所有峰之后

```
信号：峰1(300)  峰2(400)  ...  谷1(1000)
```

| 算法 | 行为 | 结果 |
|------|------|------|
| **原算法** | 谷之后无候选峰 → 从全局选最近的峰 | 强制配对 谷1-峰2 |
| **改进后** | 谷之后无候选峰 → 跳过谷1 | 返回空配对列表 |

### 场景：3 峰 3 谷，正常交替分布

```
信号：峰1(300)  谷1(360)  峰2(700)  谷2(760)  峰3(1100)  谷3(1160)
```

| 算法 | 行为 | 结果 |
|------|------|------|
| **原算法** | 每个谷找到最近的峰，强制配对 | 3 个配对 |
| **改进后** | 每个谷找到最近的峰，正常配对 | 2-3 个配对 |

> 注：实际测试结果为 2 个锁定点，因为第 3 强的谷可能因方向判断而被跳过，这是更合理的行为。

## 5. 向后兼容性

| 项目 | 说明 |
|------|------|
